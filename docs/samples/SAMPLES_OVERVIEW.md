## 📖 文檔結構說明

### 根目錄 README.md (簡潔版)

**目標:** 讓開發者在 2-3 分鐘內了解專案並開始使用

**長度:** 約 150-200 行

**內容結構:**
```markdown
# 專案名稱
[徽章]

## 簡介 (2-3 句話)
## 特性 (5-10 個要點)
## 技術棧 (主要工具列表)
## 快速開始
  - 前置需求
  - 安裝
  - 開發
  - 構建
  - 測試
## 可用指令 (表格形式)
## 詳細文檔 (連結)
## 專案結構 (樹狀圖)
## 貢獻
## 授權
```

**特點:**
- ✅ 簡潔明瞭,快速上手
- ✅ 完整的徽章和狀態指標
- ✅ 清晰的指令表格
- ✅ 實用的程式碼範例
- ✅ 連結到詳細文檔

## 🎨 文檔風格特點

### 1. 視覺化元素

- **Emoji 圖示:** 用於章節標題,提升可讀性
- **徽章 (Badges):** 顯示版本、授權、Node 要求等
- **Mermaid 圖表:** 架構圖、流程圖
- **表格:** 指令列表、配置選項
- **程式碼區塊:** 語法高亮的範例

### 2. 內容組織

- **漸進式深度:** 從簡單到複雜
- **實用導向:** 著重可執行的範例
- **完整覆蓋:** 涵蓋開發全週期
- **易於導航:** 清晰的目錄和錨點連結

### 3. 技術準確性

- **基於實際配置:** 所有內容反映真實的專案設定
- **可執行範例:** 程式碼範例都經過驗證
- **版本明確:** 標註具體的工具版本
- **連結有效:** 所有外部連結保持有效

## 🔍 範例對比

### 應用程式 vs 函式庫模板的差異

| 方面 | 應用程式模板 | 函式庫模板 |
|------|------------|-----------|
| **構建輸出** | 單一可執行檔案 | CJS + ESM + 型別定義 |
| **package.json** | 無 main/module | main, module, types, exports |
| **使用場景** | 部署運行 | npm 發布 |
| **文檔重點** | 部署、Docker、運行環境 | API、使用範例、發布流程 |
| **測試重點** | E2E 測試、整合測試 | 單元測試、API 測試 |

### 簡潔版 vs 詳細版文檔的差異

| 方面 | 根目錄 README | docs/ 詳細文檔 |
|------|--------------|---------------|
| **長度** | 150-200 行 | 400-600 行 |
| **目標讀者** | 快速上手的開發者 | 深入學習的開發者 |
| **內容深度** | 基礎指令和設置 | 完整技術細節 |
| **程式碼範例** | 基本用法 | 進階用法和最佳實踐 |
| **架構說明** | 簡單的目錄結構 | 詳細的架構圖和模組設計 |
